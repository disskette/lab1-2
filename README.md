# lab1-2 — Сортировка и поиск в электронном справочнике деловых партнёров

Учебный проект на C++ (две лабораторные работы по дисциплине «Алгоритмы и структуры данных»):

- **lab1** — сравнение алгоритмов **сортировки** массива объектов;
- **lab2** — сравнение алгоритмов и структур данных для **поиска** по ключу.

В качестве предметной области выбран электронный справочник деловых партнёров: ФИО, название организации, рабочий телефон, e‑mail.

## Модель данных: класс `Partner`

Поля: `name` (ФИО), `organisation` (организация), `phone` (телефон), `email` (почта).

Для класса перегружены операторы сравнения (`==`, `!=`, `<`, `>`, `<=`, `>=`). Сравнение лексикографическое, по порядку полей **организация → ФИО → телефон**: сравнивается первое несовпадающее поле, по нему и определяется результат (аналогично сравнению C‑строк).

## Лабораторная 1 — сортировки (`lab1.cpp`)

Реализованы и сравниваются по времени работы:

| Алгоритм | Сложность по времени | Память |
|---|---|---|
| Шейкерная сортировка | O(n²) средн./худш., O(n) лучший | O(1) |
| Быстрая сортировка (Хоара, рекурсивная) | O(n log n) средн., O(n²) худш. | O(log n) |
| Сортировка слиянием | O(n log n) всегда | O(n) |
| `std::sort` | O(n log n) | — (эталон сравнения) |

Программа:
1. читает один или несколько CSV‑файлов с входными данными (имена файлов передаются аргументами командной строки);
2. сортирует копии массива каждым из четырёх способов, замеряя время в миллисекундах (`std::chrono`);
3. проверяет, что все четыре результата идентичны (`sortCheck`);
4. записывает отсортированный массив в CSV;
5. выводит в консоль время каждой сортировки.

## Лабораторная 2 — поиск (`lab2.cpp`)

Реализованы и сравниваются по времени поиска **всех** записей с заданным ключом (поиск ведётся по полю «организация», допускаются дубликаты ключа):

| Метод | Сложность поиска | Память |
|---|---|---|
| Линейный поиск | O(n) | O(1) |
| Бинарный поиск (по предварительно отсортированному массиву) | O(log n + k) средн., O(n) худш. | O(1) |
| Красно‑чёрное дерево (`rbtree.h`) | O(log n + k) | O(n) на дерево + O(log n) стек |
| Хэш‑таблица с разрешением коллизий методом цепочек (`hash.h`), полиномиальный хэш (P = 31) | O(1) средн. на цепочку | O(n) + размер таблицы |
| `std::multimap` (эталон сравнения) | O(log n + k) | O(n) |

где *k* — количество найденных записей с одинаковым ключом.

Программа аналогично lab1 читает CSV, для каждого файла строит все структуры, замеряет время поиска в наносекундах, сверяет результаты всех методов между собой (`searchCheck`) и сохраняет найденные записи в CSV. Дополнительно выводится количество коллизий в хэш‑таблице.

## Формат CSV

Файлы без заголовка, поля разделены `;`, одна запись — одна строка:

```
Иванов Иван Иванович;ООО Ромашка;+7 (912) 345-67-89;ivanov@example.com
```

Каталоги `input1/`, `input2/`, `output/`, `output2/` в репозиторий не входят (см. `.gitignore`) — их нужно создать самостоятельно перед запуском.

## Сборка

Требуется компилятор с поддержкой C++17 (используются POSIX‑вызовы `read`/`open`/`creat`, поэтому сборка — под Linux/macOS).

```bash
# Лабораторная 1 — сортировки
g++ -O2 -std=c++17 lab1.cpp partner.cpp general_funcs.cpp -o lab1

# Лабораторная 2 — поиск
g++ -O2 -std=c++17 lab2.cpp partner.cpp general_funcs.cpp -o lab2
```

## Запуск

```bash
mkdir -p input1 output      # для lab1
mkdir -p input2 output2     # для lab2

# поместите CSV-файлы с входными данными в input1/ (для lab1) или input2/ (для lab2)

./lab1 data_100.csv data_1000.csv data_10000.csv ...
./lab2 data_100.csv data_1000.csv data_10000.csv ...
```

Каждое имя файла передаётся отдельным аргументом, программа ищет их в `./input1/<имя>` (`lab1`) или `./input2/<имя>` (`lab2`) и сохраняет результат в `./output/` / `./output2/` соответственно.

## Результаты замеров

В каталоге `графики/` приведены графики зависимости времени выполнения от размера массива:

- **сортировки_все.png / сортировки_быстрые.png** — шейкерная сортировка заметно проигрывает (рост ~O(n²)) быстрой сортировке, сортировке слиянием и `std::sort`, время выполнения которых практически совпадает на исследованных размерах;
- **поиски.png** — линейный поиск растёт линейно и сильно отстаёт от остальных методов уже на массивах от нескольких тысяч элементов; бинарный поиск, красно‑чёрное дерево, хэш‑таблица и `std::multimap` показывают сопоставимое логарифмическое/околоконстантное время;
- **коллизии.png** — статистика коллизий в хэш‑таблице в зависимости от размера входных данных.
